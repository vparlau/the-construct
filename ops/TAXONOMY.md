# TAXONOMY — Canonical Tags

Use **lowercase-hyphen** slugs. Choose the smallest set that accurately describes the artifact. These tags power search, filtering, and cross-domain analytics.

## Core categories & examples

### domain
`writing`, `research`, `customer-support`, `marketing`, `product`, `education`, `engineering`, `data-analysis`, `design`, `operations`

### capability
`summarization`, `generation`, `classification`, `extraction`, `rewriting`, `planning`, `ideation`, `critique`, `translation`, `evaluation`

### modality
`text`, `image`, `audio`, `video`, `code`, `tabular`

### output-format
`markdown`, `json`, `plain`, `csv`

### audience
`general`, `executive`, `engineer`, `designer`, `marketer`, `student`

### tone
`formal`, `casual`, `persuasive`, `instructional`, `technical`, `friendly`, `authoritative`

### complexity
`basic`, `intermediate`, `advanced`

### safety
`safe`, `sensitive`, `regulated`

## Tagging rules
- Prefer 3–7 tags total.
- Use at least one from **capability** and one from **output-format**.
- Avoid synonyms; pick one canonical term.
- If you need a new tag, add it here with a one‑line definition.
