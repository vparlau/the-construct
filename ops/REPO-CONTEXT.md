# Repository Context — Internal Addendum

This document augments the **canonical** repository context at the root (`/REPO-CONTEXT.md`) with internal clarifications for maintainers and agents.

## 1) Contract Reinforcements
- **Minimal domains:** Keep only `README.md`, `INDEX.md`, `USAGE-GUIDE.md`, `CHANGELOG.md`, and content folders; add a domain `AGENTS.md` only when specialization is needed.
- **Two-tier changelogs:** Root log = summary; domain logs = authoritative local history.
- **No-code guarantee:** All governance lives as Markdown and JSON Schemas here; automation, if any, runs outside the repo.

## 2) Front‑Matter Policy (required fields)
`id`, `title`, `domain`, `type`, `version` (SemVer), `release_status`, `owner`, `license`, `tags`, `context_budget`, `inputs.schema`, `output_expectations`, `constraints`, `safety`, `changelog_ref`.
See `ops/schemas/frontmatter.schema.json` for exact shapes.

## 3) Indexing & Search
- Human indexes: root `INDEX.md` (map of domains), domain `INDEX.md` (authoritative catalog).
- Machine index (optional): `ops/catalogs/CATALOG.json` may be generated by external tooling from artifact front‑matter.

## 4) Release Discipline (short form)
- Update artifact → update domain `INDEX.md` → append domain `CHANGELOG.md` → append root `CHANGELOG.md` → (if policy/taxonomy changed) update `ops/` docs → verify discoverability.
